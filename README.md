# yamyam — 날씨 기반 강남구 맛집 추천

> 교외 팀 프로젝트 (2인) · 담당: 검증 분석 · 추천 로직 · 성능 평가 (팀원: 데이터 수집 · 시각화)
> 데이터: 네이버 데이터랩 검색 트렌드, 기상청 ASOS, 서울 열린데이터 강남구 음식점 · 도구: Python (pandas, scikit-learn, pyproj)

## 문제

"날씨에 따라 사람들이 먹고 싶은 음식이 달라진다"는 건 흔한 통념이다.
이게 실제 데이터로도 성립하는지 검증하고, 성립한다면 그 관계로 위치 기반 맛집을 추천해보고자 했다.

## 데이터

- **검색 트렌드**: 네이버 데이터랩 — 국밥·칼국수·냉면·파전·팥빙수 5개 음식의 일별 검색지수 (2023.01.01 ~ 2025.08.31)
- **날씨**: 기상청 ASOS 서울 지점 일자료 (평균기온·일강수량·습도·풍속)
- **음식점**: 서울 열린데이터광장 일반음식점 인허가 — 강남구 필터, 영업중 + 좌표 보유 **10,932곳**
  - 좌표가 TM(EPSG:5174)이라 위경도(WGS84)로 변환해 위치 계산에 사용

## 1단계 — 날씨 ↔ 음식 선호 검증

검색 트렌드와 날씨를 날짜로 결합해 상관을 봤다 (974일).

| 음식 | 기온 | 강수 |
| --- | --- | --- |
| 냉면 | **+0.71** | -0.02 |
| 팥빙수 | **+0.73** | +0.09 |
| 파전 | +0.08 | **+0.53** |
| 칼국수 | -0.11 | +0.21 |
| 국밥 | -0.12 | -0.02 |

- **냉면·팥빙수는 기온과 강한 양의 상관** (더울수록 검색 급증)
- **파전은 강수와 강한 양의 상관** — 비 오는 날 검색이 2.8 → 5.4로 **약 2배**
- **국밥은 날씨와 사실상 무관** (통념과 달리 모든 음식이 날씨를 타지는 않았다)

## 2단계 — 추천 로직

1. 음식별로 검색지수를 날씨(기온·강수·습도·풍속)로 선형 회귀 학습
2. 입력 날씨에 대해 각 음식의 예측 검색량을 **그 음식 기준 z-점수**로 변환 → 음식 간 비교 가능하게 만들어 순위화
3. 상위 음식의 강남구 영업중 음식점을 **사용자 위치에서 가까운 순**(haversine)으로 추천

예) 덥고 맑은 날(강남역) → 냉면·팥빙수 → 설빙 강남역지점(0.36km), 동아냉면(0.41km) …
예) 비 오는 날 → 칼국수·파전 → 등촌샤브칼국수(0.15km), 윤대감 빈대떡(0.21km) …

## 3단계 — 성능 검증 (지표)

시계열 분할(2025년 이전 학습 726일 / 2025년 검증 243일)로, "검증일의 날씨로 그날 실제 가장 급등한 음식을 맞췄는가"를 측정했다.

| 지표 | 결과 | 무작위 기대치 |
| --- | --- | --- |
| Hit@1 | **58.8%** | 20% |
| Hit@2 | **79.4%** | 40% |

무작위 대비 Hit@1은 약 3배, Hit@2는 약 2배. **날씨로 음식 수요를 유의미하게 예측할 수 있음**을 확인했다.

## 한계

- **국밥처럼 날씨 신호가 약한 음식**도 있다 — 모든 메뉴가 날씨를 타는 건 아니다.
- **파전·팥빙수는 일반음식점 인허가에 이름으로 적게 잡힌다** (빙수는 카페 업종, 파전은 '한식'으로 등록되는 경우가 많음) → 후보가 얇다.
- 검색량은 **실제 방문·매출이 아니라 관심 지표**다. 실측 데이터라면 더 정확한 검증이 가능하다.
